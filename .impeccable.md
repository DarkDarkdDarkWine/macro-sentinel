# .impeccable.md — Macro Sentinel Design Context

## Design Context

### Users

单一用户（产品本身的开发者），每日开盘前或收盘后使用，目的是在 5–10 分钟内完成宏观判断：
- 今天全球风险是否升温？
- 市场处于恐慌还是平静？
- 有哪些地缘政治事件需要关注？

使用场景：坐在电脑前，精力集中，想要快速扫描关键信号，不需要被引导，直接看结论。

---

### Brand Personality

**三个词**：冷静 · 精准 · 克制

- **彭博终端**的精神：信息密度高、数据优先、功能压倒装饰
- **苹果股市 App** 的气质：数字突出、留白适度、中英文混排雅致
- 不是交易软件，是**判断工具**——界面应让人沉下心来思考，而不是制造紧迫感

---

### Aesthetic Direction

**主题**：深色唯一（不做浅色模式）

**颜色规则**：
- 红绿系统保留金融惯例：绿涨红跌，不可变动
- 背景色系延续现有深海蓝方案（`#0f1117` / `#1e293b`），有彭博终端的质感
- 强调色保持蓝色（`#60a5fa`），不引入橙色、紫色等情绪色
- 风险等级用色：绿（低）→ 黄（中）→ 橙（高）→ 红（极高），与 VIX 一致

**版式**：
- 顶部分区：宏观风险评级 + 今日核心结论（后续阶段实现）
- 下方分区：市场数据 / 宏观指标 / 新闻三列并排
- 数字用等宽数字（`font-variant-numeric: tabular-nums`）保证对齐
- 中英文混排：英文符号（$、%、指数代码）与中文标签之间保留合适间距，字体栈优先保证中文渲染清晰

**反参考（不做什么）**：
- 不做 Robinhood 风格的彩虹配色和游戏化元素
- 不做大面积渐变、霓虹发光特效
- 不做圆角过大的卡片（保持当前 10px，不超过 12px）
- 不用动效吸引眼球，动效只服务于状态反馈（加载、完成）

---

### Design Principles

1. **数据即界面** — 每一个像素要么承载信息，要么创造信息之间的间距。装饰元素只在服务可读性时存在。

2. **色彩只说事实** — 颜色是数据的编码，不是品牌的装饰。绿色 = 上涨/安全，红色 = 下跌/危险，黄色 = 警戒，灰色 = 中性/未知。不因视觉美观改变色彩的语义。

3. **结论前置，细节后置** — 最重要的判断（风险等级、今日摘要）在视口顶部，支撑数据在下方。用户不应该在找结论。

4. **克制的专业感** — 字重、字号层级清晰但不夸张。标题用全大写+宽字距建立层级，正文保持正常字重。不过度强调。

5. **中英文一等公民** — 中文标签和英文符号/数字同等重要。混排时保证视觉节奏统一，不因中文字符宽度导致布局跳动。
